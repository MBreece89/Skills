# Changelog

All notable changes to these skills will be documented here.

## 2.0.0

### Added — Workflow Skills (User-invoked)
- `ask-me.md` — Router skill: recommends the right skill for your situation
- `grill-me.md` — Relentless interview loop to resolve a plan before building
- `grill-with-docs.md` — Grilling session that also updates `CONTEXT.md` and ADRs
- `triage.md` — Issue triage state machine (Inbox → Needs Info → Refined → Ready)
- `improve-codebase-architecture.md` — Scan for architecture weaknesses, produce a report
- `setup-skills.md` — Configure skills for a new project (issue tracker, labels, docs layout)
- `to-issues.md` — Break a plan or PRD into independently-deliverable vertical-slice issues
- `to-prd.md` — Synthesize the current conversation into a structured PRD
- `prototype.md` — Build a throwaway prototype (logic or UI) to answer a design question
- `handoff.md` — Compact the session into a handoff doc for another agent or future session
- `teach.md` — Multi-session teaching with tracked progress per topic
- `writing-great-skills.md` — Reference for writing and editing skill files well

### Added — Workflow Skills (Model-invoked)
- `grilling.md` — The reusable interview loop behind `grill-me` and `grill-with-docs`
- `tdd.md` — Red-green-refactor TDD loop, one vertical slice at a time
- `diagnosing-bugs.md` — Reproduce → minimise → hypothesise → instrument → fix → regression test
- `domain-modeling.md` — Build and sharpen the project glossary; update `CONTEXT.md` and ADRs
- `codebase-design.md` — Deep module design: lots of behaviour behind a small interface

## 1.0.0

### Added — Reference Skills
- `testing.md` — Testing philosophy, structure, anti-patterns, and coverage guidance
- `security.md` — Secrets, input validation, auth, network, and dependency security
- `accessibility.md` — Semantic HTML, ARIA, keyboard navigation, forms, and media
- `documentation.md` — Inline comments, API docs, README, architecture, and changelog standards
- `git-conventions.md` — Commit messages, branching, PRs, and workflow
- `code-review.md` — Author and reviewer responsibilities, feedback standards, and anti-patterns
